## Commands

All commands run from the `slides/` directory unless noted.

```bash
# Render to HTML (the main output)
quarto render ml-intro.qmd

# Live preview with auto-reload on save
quarto preview ml-intro.qmd

# Export to PDF (requires Chrome/Chromium)
quarto render ml-intro.qmd --to pdf

# Regenerate all Mermaid diagrams after editing diagrams/*.mmd
./render-diagrams.sh        # renders + post-processes (rounds corners, thickens arrows)

# Regenerate matplotlib figures (overfitting, embeddings, latency, scaling, clf-vs-reg, attention, confusion, clustering, tokenize)
python3 figures/make-figures.py

# Audit which slides overflow the 1280├Ś720 viewport
quarto render ml-intro.qmd
node tools/measure-overflow.js "$PWD/ml-intro.html"   # needs: npm i puppeteer-core
```

## Architecture

### Single-file deck

The entire lecture lives in `slides/ml-intro.qmd` ŌĆö a Quarto Reveal.js presentation. Sections are delimited by `# Title {background-color="#7a1f2b"}` (dark-red divider slides) and slides by `## Heading`. Adding `{.smaller}` to a `##` heading reduces body font size for content-heavy slides.

### Two SVG pipelines

**1. Mermaid diagrams** (`diagrams/*.mmd` ŌåÆ `images/diag*.svg`)  
Flowcharts are authored in `diagrams/diag01.mmd` ŌĆ” `diag16.mmd` using a shared `classDef` palette and pre-rendered to static SVG by `render-diagrams.sh`. Pre-rendering avoids a Reveal.js bug where hidden slides are measured at zero width. The script also runs a Python post-processor that injects `rx="8"` on node rects and thickens arrows ŌĆö do not skip this step. Config is in `mermaid-config.json`.

**2. Hand-crafted SVGs** (`images/*.svg` ŌĆö not `diag*.svg`)  
Custom comparison diagrams (e.g. `framing-clf.svg`, `recsys.svg`, `multiclass.svg`) are written directly in SVG. Style conventions to match:
- Font: `Inter, 'Segoe UI', 'Noto Sans Thai', system-ui, sans-serif`
- Rounded boxes: `rx="8"` (nodes), `rx="10"` (panels/outer borders)
- Color palette: red `#b8232f` / `#7a1f2b`, green `#137333`, blue `#1d4ed8`, amber `#d97706`, neutral `#f7f8fa` / `#aaa`
- Arrow markers: `fill="#555"`, `stroke-width:2`
- Panel backgrounds: light tint of the accent color with a 2ŌĆō2.5px colored border

**3. Matplotlib figures** (`figures/make-figures.py` ŌåÆ `images/*.svg`)  
Scientific/data plots. Use the same color constants defined at the top of that script (`RED`, `GREEN`, `BLUE`, `MUTE`, `INK`).

### Theme and CSS

`custom.scss` defines the CMU-red theme. Key utility classes available in the deck:

| Class | Use |
|---|---|
| `.card` | Callout box with left red border |
| `.keywords` | Yellow "explore" bar at slide bottom |
| `.big` | Large lead text (1.4 em) |
| `.diagram` | Block-centers an SVG image |
| `.photo` | Rounded corners + shadow on a photo |
| `.muted` / `.red` / `.green` | Inline colour spans |
| `.placeholder` | Dashed placeholder box |

### Slide content constraints

- Viewport is **1280 ├Ś 720 px** with `margin: 0.06`. Content that overflows is clipped ŌĆö use `measure-overflow.js` to catch this.
- Images default to `max-height: 50vh`. Override with `{width="80%"}` or similar on the `![]()` tag.
- The `{.smaller}` class is the first fix for overflow; then trim text or shrink images.
- Absolute-positioned overlays (memes, decorative images) use inline `style="position:absolute; ..."` on a raw `<img>` tag placed at the top of the slide.

### Cross-references

Slide IDs are auto-generated from headings (e.g. `## Framing ŌĆö Classification {#framing-can-make-a-problem-easier-or-harder}`). When renaming a slide that is linked from another, keep the explicit `{#id}` anchor or update all `[text](#id)` links.

### Deployment

`git push` to `main` triggers `.github/workflows/deploy.yml`, which calls `quarto render` (no mermaid-cli or matplotlib ŌĆö SVGs are committed) and publishes to GitHub Pages at `https://kasemsit.github.io/261405/`. The course landing page is `index.html` at the repo root.

### What not to commit

- `*.pptx` ŌĆö original decks contain third-party copyrighted figures
- `slides/ml-intro.html`, `slides/ml-intro_files/`, `slides/.quarto/`, `_site/` ŌĆö all regenerated by CI
